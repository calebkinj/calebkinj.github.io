Category One: Software Design and Engineering

Enhancement Overview

The focus of Category One was improving the overall architecture, maintainability, scalability, and user experience of the Weight Tracker application. While the original CS360 project provided a functional foundation, several areas could be improved through refactoring, workflow redesign, and modernization of the application's structure.

Throughout this enhancement, I redesigned the authentication system, modernized the dashboard, implemented a centralized settings system, and refactored the application's data architecture into a more scalable DailyRecord model. These improvements transformed the application from a basic course project into a more organized and maintainable software solution.

Original Application Challenges

  • Login and registration workflows were tightly coupled.
  • Account management functionality was limited.
  • The dashboard provided minimal feedback and visualization.
  • User preferences and customization options were not available.
  • The original data structure limited future scalability.
  • Workflow organization became increasingly difficult as features expanded.

Authentication Redesign

One of the largest improvements was the redesign of the authentication system. The original application combined multiple responsibilities within a single workflow, making future expansion difficult. To improve maintainability, authentication features were separated into dedicated activities for login, registration, password recovery, and password reset.

Additional validation was added to improve account creation and account management workflows. Duplicate username prevention, password confirmation checks, password reset functionality, and improved user feedback created a more realistic authentication experience while maintaining the project's local SQLite architecture.

Dashboard Modernization

The original dashboard was redesigned to provide a clearer overview of user progress and improve the overall user experience. The updated dashboard introduced a cleaner layout, improved information hierarchy, and a circular progress visualization that provides immediate feedback on goal progress.

Additional improvements included enhanced navigation, better empty-state handling for new users, improved readability, and more intuitive organization of key health tracking information.

DailyRecord Architecture Refactor

To improve long-term scalability, the original weight-entry architecture was redesigned into a more flexible DailyRecord system. This refactor introduced support for multiple tracking metrics while preserving compatibility with existing application functionality.

The new architecture improved the relationship between the application's activities, database workflows, and dashboard functionality. It also established a stronger foundation for future enhancements and supported additional tracking capabilities such as calories burned.

Settings and User Preferences

A centralized Settings system was implemented to improve customization and account management. Users can now manage application preferences, switch between pounds and kilograms, reset progress data, and modify account information directly within the application.

Persistent preferences were implemented using SharedPreferences, allowing settings to remain consistent between sessions. Additional account management enhancements included username modification, password updates, validation handling, and synchronization of account data across related database tables.

Testing and Validation

Testing focused on validating all newly introduced workflows and ensuring compatibility with existing application functionality. Verification included user registration, password reset functionality, account management workflows, dashboard calculations, settings persistence, unit conversion, DailyRecord creation, and progress reset operations.

Additional testing confirmed proper navigation behavior, successful database integration, accurate dashboard updates, and reliable persistence of user preferences across application sessions.

Reflection

This enhancement significantly improved the overall quality of the Weight Tracker application. Rather than simply adding new features, the focus was placed on creating a more maintainable and scalable architecture while improving usability and workflow organization.

The redesigned authentication system, modernized dashboard, DailyRecord architecture, and centralized settings management established a strong foundation for the additional algorithms and database enhancements completed in Categories Two and Three. This experience reinforced the importance of designing software with future growth, maintainability, and user experience in mind.
