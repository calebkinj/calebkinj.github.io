Professional Self Assessment

Introduction

Completing the Computer Science program at Southern New Hampshire University has given me the opportunity to develop both my technical skills and my ability to approach software development as a complete process. Throughout the program, I worked on projects that required software design, programming, testing, database management, documentation, and communication. Looking back at the coursework and my Capstone project, I can see how each course contributed to my growth as a developer and prepared me for a career in the technology field.

Communication and Collaboration

One of the most important skills I developed throughout the program was the ability to communicate and collaborate effectively. While many projects were completed individually, several courses required peer reviews, discussions, stakeholder analysis, and professional documentation. In CS250, I learned how Agile development emphasizes communication between stakeholders and development teams throughout the software development lifecycle. In CS255, I worked on system analysis and design documents that required translating technical requirements into solutions that met stakeholder needs.

The Capstone project provided another opportunity to strengthen these communication skills. Throughout the enhancement process, I created code reviews, milestone reports, enhancement narratives, testing documentation, and project summaries. Writing these documents required me to explain not only what changes were made to the application, but also why those changes improved the overall design. This experience reinforced the importance of clear documentation and professional communication within software development projects.

Software Design and Engineering

The Software Design and Engineering enhancement focused on improving the overall structure, usability, and maintainability of the Weight Tracker application that was originally developed in CS360. The original application contained basic functionality, but several areas could be improved.

During this enhancement, I redesigned the authentication system by separating login, registration, password recovery, and password reset functionality into dedicated activities. I also redesigned the dashboard, implemented a settings system, added unit conversion between pounds and kilograms, and reorganized database interactions to support a more scalable DailyRecord architecture.

This enhancement demonstrated how software engineering principles can improve both user experience and long-term maintainability. Instead of continuing to add features to an increasingly complex codebase, I focused on refactoring and modular design. Breaking functionality into dedicated activities and helper methods made the application easier to understand, test, and expand in future iterations.

Through this process, I gained a deeper appreciation for designing software with future growth in mind rather than only meeting immediate requirements.

Algorithms and Data Structures

The Algorithms and Data Structures enhancement focused on organizing and presenting historical data in a meaningful way. After completing the Software Design and Engineering enhancement, the application had evolved significantly, creating an opportunity to improve how users interacted with their stored information.

Rather than simply displaying records in a list, I developed a calendar-based history system that generates monthly calendar views, highlights dates containing records, allows users to navigate between months, and displays detailed information for selected dates.

This enhancement required me to think carefully about how data is organized, retrieved, and presented. The application needed to efficiently retrieve records for a specific month, determine which calendar dates contained stored records, and calculate analytics such as average weight, total calories burned, and record counts.

Developing these features strengthened my understanding of data organization, algorithm design, and user-focused data visualization. It also demonstrated how thoughtful data structures and retrieval methods can significantly improve the usability of an application.

Databases and Data Management

The Databases enhancement focused on expanding the SQLite database and improving database integration within the application. The original project relied on a relatively simple database structure.

For this enhancement, I added a new motivational_messages table, implemented database version upgrades, populated the table with default records, and created functionality to retrieve and display a random motivational message whenever a daily record was successfully saved.

This enhancement strengthened my understanding of database design, schema evolution, and application integration. I worked with table creation, database upgrades, data insertion, and query execution while ensuring that new functionality remained compatible with existing application features.

The experience reinforced the importance of designing databases that can evolve as application requirements change over time.

Security and Testing

Security is another area where I demonstrated growth throughout the program and within the Capstone project. The original Weight Tracker application contained a basic login system with limited account management functionality.

During the Software Design and Engineering enhancement, I expanded authentication features by adding registration validation, password reset functionality, username management, password management, and improved input validation.

While the application is not intended for production use, these changes helped demonstrate fundamental security concepts such as user authentication, account management, and protection against invalid data entry. The project reinforced the idea that security should be considered throughout the development process rather than added as an afterthought.

In CS320, I learned the importance of software testing and validation. Writing unit tests demonstrated how structured testing improves reliability and helps identify issues before software is released.

Professional Growth and Career Readiness

Several other courses throughout the program also contributed to my development as a software engineer. In CS330, I worked with OpenGL to develop interactive 3D scenes, which strengthened my problem-solving abilities and taught me how to break large technical challenges into smaller, manageable tasks. Each course introduced different technologies and development approaches, helping me become more adaptable when learning new tools and frameworks.

Looking back at the entire Computer Science program, the Capstone project provided an opportunity to bring together everything I had learned. The Weight Tracker application evolved from a relatively simple mobile application into a significantly more robust project featuring improved software architecture, historical data visualization, expanded database functionality, enhanced security features, and a more polished user experience.

Just as importantly, the project required planning, documentation, testing, analysis, and communication skills that reflect real-world software development practices.

As I prepare to complete my degree and begin my professional career, I am excited to apply the knowledge and skills I have developed throughout this program. From software engineering and database design to testing, problem-solving, and technical communication, each course contributed to my growth as a developer.

One of the most valuable lessons I gained from the Capstone project was learning how to evaluate an existing application, identify opportunities for improvement, and implement enhancements while maintaining compatibility with existing functionality.

My goal is to continue growing as a software developer by working on projects that challenge me to learn new technologies, solve real-world problems, and contribute to meaningful software solutions. This portfolio represents not only the completion of my Computer Science degree, but also the foundation I have built for my future career in the software industry.
