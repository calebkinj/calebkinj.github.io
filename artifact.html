Original Artifact

Artifact Overview

The artifact selected for this Capstone project was a Weight Tracker mobile application originally developed in CS360: Mobile Architecture and Programming. The application was designed to help users record their daily weight, manage personal weight goals, and receive notifications related to their progress.

While the application successfully met the requirements of the original course project, it also provided several opportunities for improvement. The project contained a solid foundation that could be expanded through software engineering, algorithms and data structures, and database enhancements. Because of this, it was selected as the single artifact used throughout all three Capstone enhancement categories.

Original Features

The original application included several core features that allowed users to track their progress and manage their accounts.

  • User Login and Registration
  • Daily Weight Entry
  • Weight Goal Management
  • Basic SQLite Database Storage
  • SMS Notification Functionality

These features provided the foundation for a functional mobile application and demonstrated the mobile development concepts covered in CS360.

Original Limitations

Although the application functioned correctly, several areas could be improved. The authentication system was limited, account management options were minimal, and users had no way to recover or reset account information. The dashboard provided only basic information, and there was no way to view historical records beyond what was stored in the database.

The application also lacked analytics, historical data visualization, user customization options, and advanced database functionality. These limitations created opportunities for meaningful enhancements that aligned with the goals of the Capstone project.

Why This Artifact Was Selected

This project was selected because it provided a strong foundation while still offering significant opportunities for improvement. The application already contained working functionality, making it possible to focus on enhancing and expanding the existing design rather than creating a completely new system from scratch.

Additionally, the project supported all three Capstone enhancement categories. The application architecture could be improved through software engineering enhancements, historical record management provided opportunities for algorithms and data structures improvements, and the SQLite database offered a platform for demonstrating database design and integration concepts.
